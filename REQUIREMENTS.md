# Требования к проекту

## Функциональные требования

### Основные функции приложения

#### Сортировка объектов кастомных классов
- Реализация алгоритмов сортировки по всем 3 полям класса
- Использование паттерна Стратегия для различных алгоритмов сортировки
- Реализация сортировки с использованием компараторов для кастомных требований

#### Управление данными
- Ввод данных вручную
- Генерация случайных данных
- Загрузка данных из файла
- Валидация данных при вводе

#### Работа с коллекциями
- Создание и заполнение коллекций с использованием стримов
- Реализация кастомных коллекций
- Хранение и манипуляции с объектами класса

#### Работа с файлами
- Запись отсортированных данных в файл
- Добавление данных в существующий файл

#### Многопоточность
- Подсчет количества вхождений элемента в коллекцию
- Вывод результата в консоль

### Требования к классу

#### Класс Автомобиль
- **Поля:**
  - Мощность (int)
  - Модель (String)
  - Год производства (int)
- Реализация паттерна Builder
- Реализация интерфейса Comparable для сортировки по всем полям
- Сортировка по всем 3 полям: мощность, модель, год производства

### Алгоритмы сортировки
- Реализация быстрой сортировки (QuickSort)
- Реализация базовых алгоритмов сортировки (пузырьковая, выбором, вставками)
- Реализация дополнительной сортировки: четные числа сортируются, нечетные остаются на местах

### Интерфейс пользователя

#### Цикл работы
- Программа выполняется в цикле
- Возможность выхода только по выбору пользователя

#### Выбор способа заполнения
- Ввод данных вручную
- Генерация случайных данных
- Загрузка данных из файла

#### Настройка параметров
- Выбор длины массива данных
- Выбор алгоритма сортировки

## Нефункциональные требования

### Архитектурные требования

#### Соблюдение паттернов проектирования
- Использование паттерна Стратегия
- Реализация паттерна Builder для класса
- Правильное применение интерфейсов и абстракций

#### Архитектура приложения
- Четкое разделение на слои (представления, бизнес-логика, данные)
- Минимизация зависимостей между модулями

#### Стиль кодирования
- Соответствие Java кодстайлу
- Читаемость и поддерживаемость кода
- Использование аннотаций для документирования

### Требования к реализации

#### Сортировка
- Запрещено использовать готовые реализации сортировки
- Все алгоритмы сортировки должны быть реализованы вручную
- Реализация быстрой сортировки (QuickSort) как основной алгоритм
- Реализация дополнительных алгоритмов (пузырьковая, выбором, вставками) для сравнения

#### Коллекции
- Использование стримов для заполнения коллекций
- Реализация кастомных коллекций
- Все коллекции должны быть реализованы самостоятельно

#### Валидация
- Полная валидация входных данных
- Обработка ошибок ввода
- Проверка корректности данных при загрузке из файлов

#### Файловая система
- Поддержка записи данных в файл
- Режим добавления данных в существующий файл
- Обработка исключений при работе с файлами

#### Многопоточность
- Реализация многопоточного подсчета количества вхождений
- Потокобезопасность при работе с коллекциями
- Корректная работа с параллельными потоками

### Производительность
- Оптимизация алгоритмов сортировки
- Эффективное использование памяти
- Минимизация времени выполнения операций

### Поддержка
- Четкая структура кода для дальнейшей поддержки
- Комментарии к ключевым частям кода
- Документация к основным методам и классам
